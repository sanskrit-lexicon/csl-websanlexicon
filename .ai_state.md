  New `v02/makotemplates/web/security_headers.php` emits `X-Content-Type-Options: nosniff`,
  `Referrer-Policy: strict-origin-when-cross-origin`, `X-Frame-Options: SAMEORIGIN`, and a
  `Content-Security-Policy-Report-Only` policy — `require_once`'d at the top of the 9 real

  `webtc/servepdf.php`, `webtc1/index.php`, `webtc1/disphier.php`, `webtc1/listhier.php`,
  `webtc2/query.php`, `mobile1/index.php`) before any other output. Deliberately skipped the
  `webtc1` support/class files (`listparm.php`, `listhiermodel.php`, `listhierview.php`) —
  they're `require_once`'d by their caller *after* output has already started, so a header
  call there would just warn "headers already sent". CSP omits `'unsafe-inline'` on
  script-src/style-src on purpose (see ROADMAP Wave 3 section for why) so Report-Only
  telemetry is actually useful for planning stage 3. Verified: `php -l` on all non-Mako
  entry points, full-tree Mako `Template()` compile (35/35 OK), YAML lint — all green via
  `C:\xampp\php\php.exe`. Stage 3 (tighten + enforce) explicitly NOT done — needs live
